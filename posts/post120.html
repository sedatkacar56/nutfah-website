onclick vs addEventListener — Know the Difference

🚀 Do you know the real secret behind JavaScript's "if clauses" of onclick and addEventListener?

Let's keep it simple. In JavaScript, both onclick and addEventListener react to user actions — like pressing a button. But there is a hidden rule many beginners don't know:

🔸 onclick — only ONE block survives

If you assign two different actions to the same button using onclick, the second one deletes the first.

❌ Example (only the last one runs)

const btn = document.getElementById("myBtn");
btn.onclick = () => { console.log("Task A");};
btn.onclick = () => { console.log("Task B"); };

➡️ Result: Only "Task B" prints.
➡️ "Task A" is overwritten and disappears.

This is why onclick is considered limited.

🔸 addEventListener — run ALL blocks safely

With addEventListener, you can attach multiple if-clause-like blocks to the same button.

✅ Example (both run)

const btn = document.getElementById("myBtn");
btn.addEventListener("click", () => { console.log("Task A"); });
btn.addEventListener("click", () => { console.log("Task B"); });

➡️ Result: Task A runs
➡️ Then Task B runs
➡️ Nothing overwrites anything.

This is why modern JavaScript always prefers addEventListener.

🎯 Simple summary
👉 onclick = only 1 action allowed
👉 addEventListener = unlimited actions (multiple blocks)
