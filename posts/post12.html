Error Handling: Use tryCatch() to Keep Scripts Running

The Problem

🚫 You loop through genes to plot with Seurat::FeaturePlot()... Everything works — until one gene is missing. Boom 💥 Error. Your whole script stops.

But why stop the show for one gene?

The Solution

Use tryCatch() to catch errors and keep moving:

tryCatch({
FeaturePlot(seurat_obj, features = "Trouble_maker") |> print()
}, error = function(e) {
message("⚠️ Skipped: ", e$message)
})

💡 Key Takeaway

Don't stop on the first error. Catch it. Report it. Move on. This is essential for processing large datasets where individual failures shouldn't halt the entire pipeline.
