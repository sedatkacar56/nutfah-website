tar Broken Pipe Error — The Sneaky Extra Slash

❌ tar: Error — Broken Pipe! What did I do wrong?

Today I ran this command on the cluster:

tar -czvf ILMN_9999.tar.gz/ ILMN_9999

and got this error message:

tar: ILMN_9999.tar.gz/: Cannot write: Broken pipe
tar: Child returned status 2
tar: Error is not recoverable: exiting now

At first glance, it looked like a serious system problem — but it turned out to be a tiny syntax mistake. Can you spot it? 😅

The issue was that extra slash (/) after the output file name, which caused tar to interpret the archive filename as a directory instead of a file, triggering the broken pipe error.

The corrected command should be:

tar -czvf ILMN_9999.tar.gz ILMN_9999
