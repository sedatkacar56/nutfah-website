CellTypist: predicted_labels vs majority_voting — Which to Use?

🧬 Do you know that in CellTypist you get two annotation outputs? Before we dive in: CellTypist is a reference-based machine learning tool for automatic cell type annotation in single-cell RNA-seq data. It uses trained logistic regression models built from curated reference datasets to predict cell identities in your dataset. When you run CellTypist, you don't get just one answer — you get two: predictedlabels majorityvoting So what's the difference?

🔹 predictedlabels Each cell is labeled completely alone The model looks at that one cell's gene expression and asks "what are you?" No context, no neighbors, no smoothing Raw, independent prediction

🔹 majorityvoting Takes the predictedlabels Groups cells by over-clustering Votes within each group Corrects outliers using neighbors

🎯 Which One to Use? Always use majorityvoting. It's the final cleaned answer. predictedlabels is just the intermediate step — useful only if you want to see where the model was uncertain. If predictedlabels and majorityvoting disagree on a cell (like FB5 vs FB4activated in your data), it means that cell was an outlier that got corrected by its neighbors.

🧠 Simple Analogy predictedlabels = one doctor's diagnosis alone in a room majorityvoting = that same doctor's diagnosis reviewed by a committee of neighboring doctors who looked at similar patients

🔥 Clean. 🧬 Accurate. 🎯 Practical.
