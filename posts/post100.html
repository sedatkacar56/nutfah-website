Don't Push Everything to GitHub — Use .gitignore

🚫 Don't want to push certain files or folders to GitHub? We've all been there — your project runs great, but you don't want to upload everything. Maybe it's:

🧪 test data
🔐 config files with credentials
🗂️ large folders or even those sneaky .env files

Here's the fix 👉 use a .gitignore file!

When you create a .gitignore, you tell Git which files and folders to skip when pushing your project.

💡 Example:

__pycache__/
.env
data/
*.log

📂 Save it as: .gitignore (Important — starts with a dot, no .txt at the end!)

🪄 In Notepad: choose "All Files" in the dropdown before saving.

That's it — your GitHub repo stays clean, secure, and lightweight.
