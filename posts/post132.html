innerText vs textContent vs innerHTML

The difference between innerText, textContent, and innerHTML

All three let you read or change what's inside an HTML element, but they behave differently.

1️⃣ innerText — what the user sees
- Returns only visible text
- Ignores hidden elements
- Affected by CSS (display: none, hidden)
- Slower (browser must calculate layout)

Example: in html:

<p>Hello</p>
<p hidden>Hidden text</p>

in js:

element.innerText; // "Hello"

👉 Use it when you care about what's currently visible on the screen.

2️⃣ textContent — all text, no matter what
- Returns all text inside the element
- Includes hidden text
- Includes spacing and line breaks
- Does not parse HTML
- Faster than innerText
- Safe for user input

Example:

element.textContent; // "Hello Hidden text"

👉 Use it when you want raw text only, especially from user input.

3️⃣ innerHTML — actual HTML
- Returns HTML markup
- Parses and renders tags
- Can run malicious code if misused
- Useful for inserting known HTML templates

Example: In js:

element.innerHTML; // "<p>Hello</p><p hidden>Hidden text</p>"

👉 Use it only with trusted content.

⚠️ Security warning (important)

Never do this with user input: In js:

element.innerHTML = userInput; // ❌ dangerous

Always do this: In js:

element.textContent = userInput; // ✅ safe

One-line memory trick
- innerText = what you see
- textContent = all text
- innerHTML = real HTML
