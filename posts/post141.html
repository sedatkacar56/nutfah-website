Python Is a Weird Language — How Functions End Across Languages

PYTHON IS A WEIRD LANGUAGE! 🔚 How functions end in different programming languages!!! Have you ever compared?

🟦 Group 1: Curly braces { } define the end
👉 Block starts with { and ends with }
👉 Indentation is only for humans
Languages: R JavaScript C C++ Java Go C# PHP

Example (all same idea):

function f() {
  do_something();
}

📌 Compiler ignores indentation completely.

🟩 Group 2: Keywords define the end
👉 Block starts with a keyword and ends with another keyword
Languages: Bash / Shell Ruby MATLAB / Octave Lua Fortran SQL (procedural)

Examples:

-bash-
if condition; then
  echo "hi"
fi

-ruby-
def f
  puts "hi"
end

📌 Very explicit, very readable.

🟨 Group 3: Indentation defines the end
👉 No braces, no end
👉 Whitespace IS syntax
Language: Python (almost unique)

Example:

def f():
    print("inside")
# dedent → function ends

📌 Misalignment = syntax error or logic bug.

This way of python coding remind me back in the days the aesthetic writing in primary school days.

Long story short, Python forces good formatting, prevents hidden bugs, and makes what you see exactly what the program runs.
