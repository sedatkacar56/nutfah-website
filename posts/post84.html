🚀 Regex in R and Single-Cell Analysis

🔹 What is Regex? Regex stands for regular expression. It's a language of pattern; a set of rules we can use to find, extract, replace, or split text. Think of it as a Swiss-army knife for text data. Why is the Regex, a pattern language, important for a single cell guy? Because we use it indirectly in our analysis, search, algorithms, functions etc. Let me remind you a little bit:

a-Percentage of specific gene group:

seurat_obj[["percent.mt"]] <- PercentageFeatureSet(
  seurat_obj,
  pattern = "^MT-"
)
seurat_obj[["percent.ribo"]] <- PercentageFeatureSet(
  seurat_obj,
  pattern = "^RP[SL]"
)

b-Metadata search:

grep("^percent", colnames(seurat_obj@meta.data), value = TRUE)
# "percent.mt" "percent.ribo"

c-Feature Name Cleanup

gsub("-[0-9]+$", "", rownames(seurat_obj))
# removes trailing -1, -2, etc.

✅ Takeaway: Regex in single-cell is everywhere: from gene QC (mito/ribo/hemo), to metadata wrangling, to barcode filtering and gene family analysis.

Deep Note: You do not need to memorize every regex command. When the right time comes, and that as you use over and over , you will get the hang of it. However, this post is for raising awareness and building mindfulness how we use regex in single cell RNA.
