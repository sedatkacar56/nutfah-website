🚀 Why Your Imported Functions Sometimes Don't Highlight in VS Code If you work with ES6 modules, you're already used to the cycle: export a function → import it where you need it → use it in your logic. But have you ever noticed something strange? You import a function or variable… and VS Code doesn't highlight it. No error. No warning. But the color is faded, almost "inactive." Why? Developers often assume: "Maybe I defined this function twice?" "Maybe the function doesn't exist in the original file?" "Maybe VS Code is glitching?" None of the above. 👉 The real reason is simpler than you think: The function (or variable) is imported but never used in that specific module. VS Code automatically dims unused imports to help keep code clean. It's a subtle visual reminder: "You brought this into the file, but you never actually called it."
