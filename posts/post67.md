- / Have you ever heard of AWS S3 bucket and AWS EC2? AWS = Amazon Web Services. It is the platform that supports data storage into buckets (S3) and cloud computing (EC2). 3 in front of S refers to 3 S's: Simple Storage Service 2 in front of EC refers to 2 C's: Elastic Compute Cloud - / First time I came across this system was when downloading FASTQ, mask, and matrix files of spatial transcriptomics — almost 2 TB of data. At that time, the company had provided a manual on how to download the data step-by-step from their S3 bucket. They also let us know we had one shot to download the files. A/ ☁️ AWS for Bioinformatics If you're wrangling .bam, .rds, or FASTQ files, this is your cloud backbone. Think of it as a global, pay-as-you-go HPC cluster — built for scale. B/ 🔹 S3 (Simple Storage Service) Your terabyte-sized Dropbox — but programmable. You can simply copy by: bash >aws s3 cp results.rds s3://my-bucket/ That's it. One command. Done. 🔹 EC2 (Elastic Compute Cloud) Your lab's virtual HPC node. Boot up RStudio in 2 minutes. Run. Shut it down. No idle charges. 🔹 CLI (Command Line Interface) No clicking. Just Linux terminal power. Shell folks — you're already home. C/ 📦 How Much Can You Store in Amazon S3? 🔹 Total Storage: Unlimited 🔹 Max File Size: 5 TB (use multipart upload if >5 GB) 🔹 Buckets per Account: 100 (soft limit) 🔹 Objects per Bucket: Unlimited 💡 S3 scales effortlessly — whether it's a few .txt files or terabytes of genomics data. D/ There are 2 user types in the AWS system: IAM vs. Root Root can do everything — it's the administrator. IAM is a guest account — only allowed to do what the root user permits. ✅ Use IAM users ✅ Limit access (S3-only, EC2-only, etc.) ✅ Share access without sharing your whole kingdom Why it matters: • Safer collaboration • Audit trails that make sense • You stay in control, not in everyone's way 💬 Want a walk-through of S3 setup or IAM roles?
